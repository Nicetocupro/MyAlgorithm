## [56\. 合并区间](https://leetcode.cn/problems/merge-intervals/)

const vector&lt;int&gt;& 传参比直接vector&lt;int&gt;传参开销要小的不少，因为const&只传递了指针 指向原数组，但是vector&lt;int&gt;每次要做三件事：

- 1\. 在堆上 malloc 新内存
- 2\. 把元素逐个复制过去
- 3\. 函数结束再 free

`const` 的作用是**承诺不修改原数据**，让编译器放心优化，同时防止意外修改。![Freehand Drawing.svg](:/70075e339e884095a7d5e8e467b884b1)

## [189\. 轮转数组](https://leetcode.cn/problems/rotate-array/)

这个有很多种方法，我的方法是使用stl的迭代器，将数组切割重组，使用std::make_move_iterator 将nums的数组分割成两份进行然后将后面那份移动到前面即可

### 我的思路：「切割重组」

```
原始: [1, 2, 3, 4, 5, 6, 7],  k=3

步骤1 - 截取后k个: nums1 = [5, 6, 7]
步骤2 - 删除后k个: nums  = [1, 2, 3, 4]
步骤3 - 插入到头: nums  = [5, 6, 7, 1, 2, 3, 4] ✅
```

**直觉清晰，但代价高：**

- 额外分配了 `vector<int> nums1`，空间 O(k)
- `erase` 会移动元素，`insert` 又移动一次
- 总体时间 O(n)，但**常数大**，有多次内存操作

* * *

### 别人的思路：「环形替换」

核心观察：旋转 = **每个元素最终去到 `(i + k) % n` 的位置**。

```
原始: [1, 2, 3, 4, 5, 6, 7],  k=3, n=7

从 i=0 出发，沿着 +k 跳：
0 → 3 → 6 → 2 → 5 → 1 → 4 → 0 (走了7步，回到起点，覆盖所有元素)
```

但不是每次都能覆盖所有元素，这取决于 `gcd(n, k)`：

```
n=6, k=2, gcd=2 → 会形成2个独立的环：
  环1: 0 → 2 → 4 → 0
  环2: 1 → 3 → 5 → 1
  各自要单独处理，所以外层循环 cnt = gcd(n,k) 次
```

**为什么是 gcd？** 从位置 `i` 出发，每次跳 `k`，回到 `i` 需要 `n/gcd(n,k)` 步， 共形成 `gcd(n,k)` 个独立的环。

**swap 的技巧：**

```
int val = nums[i];   // 暂存起点的值

do {
    int next = (cur + k) % n;
    swap(nums[next], val);  // val 始终携带"待放置"的值
    cur = next;
} while (cur != i);
```

```
以 i=0, k=3 为例，val=1(nums[0])：

cur=0, next=3: swap(nums[3], val) → nums[3]=1, val=4
cur=3, next=6: swap(nums[6], val) → nums[6]=4, val=7
cur=6, next=2: swap(nums[2], val) → nums[2]=7, val=3
...直到 cur 回到 0
```

| **性能** | O(n)，常数大 | O(n)，常数小 |

### 你需要学到的东西

**1\. 用 gcd 分析循环节** 当问题涉及「以步长 k 在长度 n 的数组里跳」时，`gcd(n,k)` 决定了独立环的数量，这是数论的基本工具。

**2\. 原地操作优先于分配新容器** 你的代码逻辑完全正确，但面试/竞赛中 O(1) 空间是更优解。遇到数组变换，先问自己：**能不能不开新数组？**

**3\. 用"携带者"变量做链式 swap**

```
int val = nums[i];  // val 像接力棒，沿着环传递
swap(nums[next], val);
```

这个模式在「置换群」类题目里很常见，值得记住。

**4\. 你的代码有一个可以改进的小细节**

```
// 原来
if(k >= nums.size()) { k = k % nums.size(); }
if(k == 0) { return; }

// 更简洁（合并处理，且防止 size()==0 时除零）
int n = nums.size();
if (n == 0) return;
k = k % n;
if (k == 0) return;
```
